# Avid Product Knowledge Base (Internal)

Private, structured reference for Avid products and versions from 2021 onwards.

This repository is designed as a **machine-readable, version-controlled source of truth**
for analysing Avid product capabilities, changes, and limitations over time.

---

## Scope

### Products covered
- MediaCentral family
  - Cloud UX
  - Production Management
  - Newsroom Management
  - Data Receiver
  - Web Services API (WSAPI)
  - Media Analytics
- Media Composer
- Avid NEXIS
- Stream IO
- Avid Content Core (ACC) — including in-product help documentation and CuttingRoom workflow transcripts

### Supplementary material
Selected transcripts from:
- NAB sessions (2025 and 2026)
- Product webinars (MediaCentral, Stream IO, Wolftech)

---

## Structure

- `_meta/`  
  Product-level metadata, compatibility matrices, virtualisation guides, failover guide, and software lifecycle reference

- `meta/`  
  Legacy metadata folder — contains `products.yml` only; not actively maintained

- `<product>/<module>/<version>/`  
  Version-specific documentation, primarily `new-features.md`, `readme.md`, and full guides

- `acc/`  
  Avid Content Core material, subdivided into:
  - `acc/help/` — in-product help documentation extracted from docs.avid.com/acc (version 0.0.3; 11 files covering overview, system requirements, sign-in, admin, user settings, UI, player, search, My Collections, Review and Approval)
  - `acc/press-releases/` — product press releases
  - `acc/resource-center/` — resource-centre articles
  - `acc/cuttingroom-transcripts/` — CuttingRoom workflow tutorial transcripts

- `transcripts/`  
  Curated transcripts relevant to news and broadcast workflows, organised into:
  - `nab/` — NAB conference sessions (2025, 2026)
  - `product-webinars/` — product webinar recordings
  - `workflows/` — workflow-specific transcript collections (e.g. CuttingRoom; note: CuttingRoom .md transcripts are also mirrored under `acc/cuttingroom-transcripts/`)

---

## Conventions

- One folder per **headline version**  
  e.g. `2024.10`, `2023.12`, `2025.6`

- Patch span (e.g. `2024.10.0–2024.10.8`) is documented in the **front matter**
  of each `new-features.md`

- All documentation is written in **Markdown**
  and assumes an **Avid-savvy, technical audience**

- `versions/current/` (Cloud UX only): contains a non-version-pinned snapshot of the most recently processed Cloud UX documentation. Treat as indicative of the latest processed version; always prefer an explicitly versioned file when one is available.

## Indexing & Usage (Important)

This repository is intended to be consumed in **stateless analysis environments**
(e.g. AI tools, scripted analysis, project-specific research contexts).

### Authoritative index

- `MANIFEST.txt` provides the **complete and authoritative list of files**
- Consumers MUST use the manifest rather than assuming directory traversal
- `MANIFEST.txt` is **curated and annotated** — it includes metadata such as source PDF names, added dates, version bounds, and flagged source anomalies. It is **not** auto-generated by a directory scan and should not be regenerated with a bare `find` command, as doing so would destroy all annotations.
- When updating the MANIFEST after adding or removing files, edit it directly and preserve the annotation format used in existing entries.
