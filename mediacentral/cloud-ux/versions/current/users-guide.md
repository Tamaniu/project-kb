(cid:129) Integration Overview (cid:129) The Story Designer - Overview of Story Designer - Opening Story Designer - Adding Production Cues to a Script Integration Overview The Story Designer is a MOS-based Web-control plug-in that enables seamless integration between ASTRA Studio, a production automation system by Aveco, and Avid’s MediaCentral Platform.
ASTRA Studio combines master control and news production automation to control both studio and production equipment, including graphics, camera robotics, video and audio mixers, video servers, lights, and more. It uses a bi-directional MOS gateway to exchange rundowns, story and clip status, clip metadata, and template information with the newsroom control system.
Administrators create templates that staff can then use to better define how stories are aired.
Journalists and producers can also exert even finer control by entering various control parameters into the stories directly from within the MediaCentral Cloud UX interface. Through the templates and optional parameters, stories are transformed into control events that manage the broadcast equipment.
The following illustration depicts a typical workflow and the systems involved in this type of production automation environment.
The Story Designer In this example, the editorial and backend systems include Story Designer with Avid’s MediaCentral core products, like Production Management, Newsroom Management, MOS Gateway, etc. The studio includes both Avid systems, like FastServe, and Aveco systems, like Astra Studio services and user interface.
The Story Designer Before using the Story Designer, you must install the plugins and complete some system setup and configuration. For installation and configuration procedures, see “Aveco Integration Setup” on page 16.
Overview of Story Designer The Story Designer is a core app (plugin) that may be docked within MediaCentral Cloud UX and used to access templates and control parameters for scripts.
> **Note:** This guide references elements of the MediaCentral Cloud UX user interface, such as the Fast Bar
and Timeline, or other core apps opened within Cloud UX, such as Search and Rundown apps. Users are expected to have a good working knowledge of the Cloud UX software. Otherwise, please see the Avid MediaCentral | Cloud UX User’s Guide for more information.
Through the Story Designer, journalists, editors, and news producers can select from a variety of production cues, refine machine controls in the app, and insert them directly into a show’s scripts.
The automation parameters that appear in the app depend on the type of production cue selected. This makes for a highly flexible user interface since parameters are largely based on the types of equipment being controlled, such as character generators, video servers, lights, cameras, and so forth.

HTML5 The Avid and ASTRA integration supports the MOS Web Control standard that is based on HTML5.
HTML5 is a modern, powerful technology that can ensure long-term and full-featured plugin support inside of systems like the MediaCentral Newsroom Management system, one of the backend systems for the browser-based MediaCentral Cloud UX user interface. HTML5 is the technology used to build the web-based user interface, while MOS is the format of the XML string of inserted items.
To use HTML5, some server-side and client-side configurations are required, but once configured, it may be used system-wide without any additional installation requirements. Configuring the Newsroom Management System Configuring the Newsroom Management System To ensure a successful integration, some configuration is required on the MediaCentral Newsroom Management system:
(cid:129) Create and Configure a Monitor Server per Show (cid:129) Create an Entry in the SYSTEM.MAP Story (cid:129) Edit the SYSTEM.MOS-MAP story (cid:129) Edit the mosconfig.xml File Create and Configure a Monitor Server per Show Monitor servers are utility programs—running on the Newsroom Management system—that check shows’ event requests for errors and send playlists to broadcast control systems like Astra Studio.
A monitor server must be assigned device and mailbox numbers—typically, these numbers are the same, which makes the configuration easier to remember. A device number is chosen for a monitor server so Newsroom Management recognizes the server (utility) program as a valid device. The device number must be entered in the system’s configuration file. The mailbox number must be assigned to both the show’s monitor server and its rundown queue.
A mailbox is an activation mechanism for a server (utility) program, so if a queue has a mailbox number matching a server program, then that server is the one activated or “awakened” whenever something happens to the queue.
