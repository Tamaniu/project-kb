# Communication and Collaboration

## Communication Top Bar

Keeping track of all your activity and communications is easy via the communications bar. All types of communications can be seen in the app top bar as well as on the mobile app.

The lighting icon is for listing and managing the alerts. The second is for tracking your bookings plan. The next one is for the activity feed (editorial updates to followed content). The last one is for notifications (Direct notifications that relate to you).

When expanding any of these buttons you will see the list of notifications. Clicking on one of them will direct you to the right place where the alert, notification, or activity was registered so you can see it in context and reply or take action immediately. The area will be highlighted in blue for easier navigation.

Note: Your settings for notifications can be set in my page setup, please review that section for further details. You can also get to them via the notification 3 dot menu.

## Chat

You will find message threads in:

- Story Message thread – The stories or productions for general conversations about the project
- Editorial segment (Background information) thread
- Booking Message Thread
- Episode (Long form) Task Message thread
- Guidance Review Message Thread – This will be logged (See guidance documentation)
- Publishing points (Output) Message thread – for delivery specific conversations

### @mention

To select the person you want to tag. They will be notified of such a message in whichever way they prefer (Email or in app). Hit Enter to Select the Resource you want to tag on a message. Do note: When @mentioning be able to search on any part of the name in any order. Show the last 25 names or groups.

### @Here

On any message threads (Production, task, booking, etc.) will notify every user in the project of your comment.

### Description Tags

@Tag someone in a description or background information field to be specific and collaborate on it is also an option.

### Breaking Paragraphs

Use Hit Shift Enter to allow you to break paragraphs on the same message.

### Chat Links

Clickable Links on chats and fields get you there faster. Great in combination with system item links where you can direct people to a specific story, segment, or output.

### Threaded Replies

Use the button underneath to Reply in Thread.

### Editing & Deleting Messages

Edit or delete a message that you have made. The deleted message will get the status deleted.

### Drafts

If you start typing a message and you don't send it because you need to check something else in another section of the system you can come back to the thread later on that day and your draft will continue to be there.

### Emoji - Special Characters

Emojis and special characters are supported as part of the comment body of text.

### Like - Acknowledgement

Acknowledgement of a chat message or a segment of text with the like button.

### Copy Message

Ability to copy a comment with the 3-dot menu.

### Drag Messages to the Story - Production Board

To highlight a piece of a conversation to the team.

## Activity Log and Feed

### The Activity Log

Tracks editorial updates on stories, pitches, logs, events, publishing points and episodes.

Changes tracked include:

a) Adjusting the description of a story or episode
b) Changes to the content Description
c) Adding, modifying or deleting a background information editorial segment
d) Adding, modifying or deleting a comment to your story or episode
e) Adding elements on the board
f) Changes to Story, episodes, Publishing Point and Editorial Segment Status and the new status is included in the notification
g) Changes to the Publishing Point Content (Template)
h) Logged changes in the additional information of Publishing Points

The Log can be found on the right panel of the item and clicking on any logged activity will direct you to where the action is happening.

The log can be filtered by the type of activity so you can find the most relevant activity quickly.

Note: For Story status changes, depending on your greenlight workflow, you may automatically change the story status to the next on greenlight, and that will log it. If your workflow does not move the status on greenlight it will log it as "ordered."

### Editorial Logs

It is important to track how stories evolve editorially so colleagues can see when new updates are available on stories. Your colleagues may be following your story so it is helpful to understand how your actions will automatically update them.

When you add a new segment, we won't notify anyone of an empty text as there is no content to share.

When you start writing inside of a new text box it will go into draft mode by default, meaning others will be able to see that you are writing something new, but they won't be able to see its content or edit it until you save it.

Editorial Segments automatically create a log on your first save. However, for further edits you get to choose to notify the activity or not, so when you are doing spell checks or minor changes that are not worth sharing no one is interrupted by unwanted prompts. The options are to save (which will notify), Save without logging, or cancel.

You can also update your colleagues following the activity of this story with the 3-dot menu: Log Activity.

### Publishing Log

Publishing points will only send updates of changes in the editor template or additional information if you press the notify button when you want to share the changes to the rest of your colleagues.

### New Activity Indicators

Across the system we are tracking when there has been recent activity on the items you are looking into since you last accessed them.

This is visible by an orange triangle indicator on the top right, which if you hover over you will see the last update to the item.

### Follow Actions

You can now easily be kept up to date with the editorial progress of the stories, pitches, events, logs, episodes, and publishing points. The follow action will subscribe you personally to be notified of the items that are tracked in the activity log (Changes across Description, Background editorial segments, Board activity, Comments, or its overall status).

Once you click on it you will be able to see the Following status.

The Follow action is also available via the mobile app.

### Activity Feed

The activity Feed is a panel that can be pulled on the top right and indicates where there have been new activities logged.

If you click on it, you can expand and minimise it like you do with the notification panel.

Clicking on one update card will take you straight to where the update happened. Whether it is highlighting a new asset uploaded to the board or a new update on the editorial section, comments etc. The aim here is to move fast to where you can take action.

For Picture uploads do note that you can just hover over the update to preview the image instantly without having to navigate to the content right away.

If you have many notifications, it is easy to mark them all as read or to unfollow them all from both web and mobile.

You can also save an update to read it later by hovering over one update and clicking on the bookmark item. This will place it on the saved tab so you can mark the rest of them all read and move at speed.

On the saved tab it is possible to remove a single item or remove all saved items.

In the updates tab, you will also notice an eye icon when you hover over the update. This allows you to focus the updates on the content related to this update and see the activity log for it.

It is then easy to use the arrow left button to come back to the global activity feed.

Above the updates you will notice a dropdown filter. This allows you to narrow the updates you see below it by type.

Next, you will see a second tab where you can see all Followed content. In the followed content section, you can get an overview of all of the content you are following to quickly manage if you want to continue to follow some of it.

As per the Updates tab if you click on the eye icon you can see the activity log for one of the items in your list but now you also have a Minus button next to it to easily unfollow that content.

The activity feed panel can be docked to the right if users have enough screen real estate. Although the default mode for the activity feed panel is floating, docking allows users to see live updates to activity across their followed stories whilst they continue to work in stories or any other view in the system.

To dock and undock simply click on the dock and undock buttons on the activity feed. This will dock it to the right so you can work on other items whilst the latest updates continue to appear side by side.

## Notifications

At the top of the communication bar you will find the notifications pane. Here it will list all of the direct communications. Whether you have been mentioned in a message, booked on a story, your project has been greenlit, etc.

You can also decide whether you also want to receive notifications on the activity updates or just track them in the activity feed.

The default when following an item is to subscribe to notifications but if you just want to see the logs in the activity feed click again on the following button to then disable notifications. This will keep you following the item in the activity feed, but notifications will not be triggered.

All of the notification preferences can be set in my page – notifications panel. Here you can also decide to receive certain notifications via Browser, In App, Email or Mobile push notifications from where you can be taken straight into the action.

## Alerts

Alerts are a straightforward way to make sure your organisation, or part of it, is aware of an event occurring. Alerts can come via Feeds or via manual alerts.

### Creating Alerts

New alerts can be created from the Alert panel that be accessed via its button. From the alert panel, it is possible to create alerts not tied to a specific item in the system.

Click to create a new alert and simply select the template you want to use and fill it out.

Alerts can be sent via the 3-dot menu from Stories, Pitches, Logs, or Events. Support for Productions is incoming.

When sending an alert, you will have the option to send it via email or as an internal Feed:

- If email you can define the emails to distribute it to, and use the email groups for it
- If it is a "Feed" alert you can define the user group to send it to or the individual users that will receive it.

### Checking Alerts

The users will see the alerts stacking up as per below on web or in the mobile app. Click on them to display their information.

If the alert comes from a story, event, or background information item it will contain the link so you can access it straight away.

Don't worry about the alert bars disappearing. Active Alerts can be seen in the Alerts Notifications Panel. And all historical alerts can be seen in the Alerts bottom left pane.

They could be displayed as well on the Feed page if configured to track them in a column.

When Alerts are sent as email from a story or event or background information editorial item you will receive an email notification with a link to the content with the information in the alert.

### Sending Alerts to 3rd Party Systems

Alerts can be configured with professional services support to be sent to 3rd party systems via Workflow Engine. You may choose for example to send them as stories to iNEWS or another NRCS or CMS system. The alerts will include all of its information including the reference object ID as well as a link to the object you send the alert from (Story, Segment, Publishing point etc).

### Non-Critical Notifications

In the communication bar you can select if you want to receive notifications for alerts. This means receiving notifications for Non-Critical Alerts as everyone receives critical alerts despite this setting. This is really for people following all wires actively and can result in many notifications coming in.

## Admin

You can set up the time out rule for when the alert bars will disappear for critical vs non critical alerts (knowing that they will always be visible in the alert panel).

There is a customer setting on the admin section to define how many critical alerts a customer wants to see.

The internal Alerts Admin section allows you to set up the different alert templates the users will see. This includes the name, giving it a unique ID if it needs to trigger 3rd party systems, making it possible to create a feed for it and the elements necessary for the alert.
