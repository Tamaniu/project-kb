Qualified Servers and Operating Systems Avid performs extensive testing on specific hardware and/or software to verify full compatibility with Data Receiver. Those that pass these tests are considered qualified by Avid.
Microsoft Windows Data Receiver Server is supported for use on Windows Server 2022 and Windows Server 2019. Data Receiver Administrator Application is additionally supported for Windows 11.
This version of Data Receiver is not supported on any other Windows operating system.
Newsroom Management Data Receiver is qualified and supported for use with Newsroom Management 2022.12 and later. It is backward compatible to previous versions of the newsroom system software.
Install and Upgrade Notes and Issues This section contains information related to installing and/or upgrading Data Receiver, and its two components: the Data Receiver Server and the Data Receiver Administrator Application.
Upgrading from Older Versions The server installer will check for older versions of Data Receiver and would not proceed with installation unless the older version is uninstalled. User should uninstall the older version through Control Panel > Programs > Programs and Features.
Prerequisites for Installation This version has dependencies on Microsoft Visual C++ redistributable which is installed with the setup (also available at this link https://aka.ms/vs/17/release/vc_redist.x64.exe ) and .NET 6.0 which could be downloaded with this link ( https://dotnet.microsoft.com/en-us/download/dotnet/thank-you/sdk- 6.0.402-windows-x64-installer).
Installing the Server Service The Server Service Installer is available under the folder DataReceiver\Server in the MediaCentral | Newsroom Management release ISO – run the setup to install the Server Service.
Install and Upgrade Notes and Issues Installing the Administrator Application The Administrator Application Installer is available under the folder DataReceiver\Administrator in the MediaCentral | Newsroom Management release ISO – run the setup to install the Administration Application.
This version of Administrator Application can be deployed on machines separately from the Data Receiver Server to enable user to monitor and maintain wires services on the Data Receiver Server.

Remote client To enable the Administrator Application to run remotely, certificates needed to be installed and firewall settings needed to be changed as outlined below. Refer to the Data Receiver Installation and Administration Guide for more detailed instructions.

Firewall setup to enable remote Administrator Application access To allow remote connection of the Administrator Application, the port 7145 must be open on the Server to accept connection request.
To open the port:
1. Click Control Panel > System and Security > Windows Defender Firewall.
2. Create a new rule to allow request on port 7145 with the following steps:
a. Select Advanced settings on the left panel.
b. Select Inbound Rules on the left panel and New Rule on the right panel to create a new rule.
This will open the New Inbound Rule Wizard.
c. On the Rule Type page, select Port.
d. On the Protocols and Ports page, select Specific Local Port and enter the number 7145; then, click Next.
e. On the Action page, select the Allow the Connection option; then, click Next.
f. On the Profile page, specify the Profiles for which this rule applies. Usually Domain & Private would be sufficient, and then click Next.
On the Name page, give this rule a name (such as Data Receiver Administrator connection) and a description, and then click Finish.
Now the port is open for communication.
